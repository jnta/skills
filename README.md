# AI Skills Ecosystem

This repository contains a specialized suite of AI agent skills designed to automate and enforce a highly structured, phased software development lifecycle. These skills operate together as an "AI Agent Assembly Line," translating high-level architectural decisions into strictly scoped, quality-gated code modifications.

## Skill Overview

The ecosystem is composed of the following core skills, each responsible for a distinct phase of the development pipeline:

### 1. `setup-skills` (The Bootstrapper)
*   **Purpose:** Idempotently initializes a workspace for AI agent compatibility.
*   **Action:** Creates the foundational `doc/AGENTS.md` rulebook and automatically chains into `quality-gate-architect` to establish repository guardrails.

### 2. `quality-gate-architect` (The Meta-QA)
*   **Purpose:** An elite DevOps meta-skill that acts as the guardian of code quality.
*   **Action:** Auto-discovers the repository's existing linters, formatters, and test frameworks, resolving gaps with the user. It then dynamically generates a custom runtime `quality-checker` skill specifically tailored to the project.

### 3. `write-a-skill` (The Meta-Builder)
*   **Purpose:** The foundational blueprint for creating new agent capabilities.
*   **Action:** Guides the creation of new skills (like the one generated by `quality-gate-architect`), enforcing a strict structure (`SKILL.md`, progressive disclosure, descriptive metadata) so the agent router can seamlessly discover and utilize them.

### 4. `adr-architect` (The System Designer)
*   **Purpose:** Stress-tests technical decisions before any code is written.
*   **Action:** Conducts a rigorous interview ("The Grilling Phase") with the user to explore edge cases and trade-offs. Once consensus is reached, it automatically synthesizes a standardized Architecture Decision Record (ADR) into `doc/adr/`.

### 5. `task-manager` (The Project Planner)
*   **Purpose:** Bridges the gap between architectural theory and actionable work.
*   **Action:** Ingests the Markdown ADRs generated by `adr-architect` and compiles them into strictly ordered, machine-readable JSON pipelines saved in `doc/tasks/`. These pipelines define atomic tasks, file scopes, and phase gates.

### 6. `code-modifier` (The Executor)
*   **Purpose:** The precise, automated code editor.
*   **Action:** Consumes individual tasks from the `task-manager`'s JSON pipeline. It executes isolated modifications strictly bounded by the task's `target_files`. Crucially, it triggers the `quality-checker` at the end of each phase to ensure the quality gate is passed before advancing.

---

## How the Skills Complement Each Other (The Automated Pipeline)

These skills are not isolated scripts; they form a cohesive, end-to-end continuous integration loop driven by local AI agents. The workflow follows this exact sequence:

1.  **Initialize:** You run **`setup-skills`** on a new project. It writes the rules and calls **`quality-gate-architect`** (using the principles of **`write-a-skill`**) to build the project's custom `quality-checker`.
2.  **Architect:** When a new feature is needed, **`adr-architect`** interrogates you and outputs an accepted ADR.
3.  **Plan:** The **`task-manager`** automatically parses that ADR and generates a JSON execution pipeline, breaking the architecture down into atomic, phased tasks.
4.  **Execute & Verify:** The **`code-modifier`** steps through the JSON pipeline task by task, altering code. Upon finishing a phase, it triggers the `quality-checker` to validate the work. If it passes, the next phase begins.

This ecosystem enforces the project's core philosophy: high-friction cognitive design (handled by the user and `adr-architect`) followed by low-friction, automated clerical execution (handled by `task-manager` and `code-modifier`).
