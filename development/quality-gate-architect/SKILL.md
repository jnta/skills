# Quality Gate Architect (Meta-Skill)

## Purpose
An elite DevOps and QA architect meta-skill. It systematically auto-discovers a repository's testing, formatting, and linting architecture, resolves missing pieces through minimal user confirmation, and uses the `write-a-skill` specification to generate a new, fully tailored runtime verification skill.

## Workflows

### 1. Pre-Flight Checklist & Existence Check
Before doing any analysis, check if a skill named `quality-checker` or a quality gate runner already exists in the workspace.
*   **If found:** Halt immediately and present an explicit warning to the user asking if they wish to proceed and overwrite the existing quality setup. Do not continue unless explicitly greenlit.

### 2. Deep Codebase Context Discovery
Scan the repository root and configuration files silently. Do not ask questions yet. Look for indicators of the quality stack:
*   **Languages & Ecosystems:** Inspect `package.json`, `pyproject.toml`, `go.mod`, `Gemfile`, `Cargo.toml`, etc.
*   **Linters/Formatters:** Check for `.eslintrc`, `biome.json`, `.prettierrc`, `ruff.toml`, `.flake8`, `black` setup.
*   **Test Frameworks:** Detect configurations for `jest`, `vitest`, `pytest`, `playwright`, `stryker.conf.js`, `mutmut`.
*   **Enterprise/Static Analytics:** Search for `sonar-project.properties` or architectural linting baselines.

### 3. Targeted Gap Confirmation
Based on the discovery step, compile a list of what was found and what is missing. Present a brief summary and ask **only** about unresolved ambiguities or critical omissions one question at a time (e.g., *"I detected pytest and ruff, but found no mutation testing framework or SonarQube properties. Are there specific execution targets or minimum coverage percentages you want to enforce as blockers?"*).

### 4. Skill Bundle Generation
Once context is completely resolved, invoke the logic of the `write-a-skill` blueprint to output a complete, decoupled skill named `quality-checker` into your skills directory.
